Motion Guidelines

How Guaji moves. Motion exists to clarify state — never to decorate. Speaking, listening, thinking, and rewards each have a signature animation; everything else uses the standard ease-out at base duration.

Principles

Do — animate state changes: AI started speaking, mic is hot, answer was correct.
Don't — animate things that don't change state: idle decoration, hover wobble, parallax for parallax's sake.
Do — keep durations between 120ms and 400ms for UI; longer only for character (mascot) and rewards.
Don't — block input. If a transition lasts >250ms, the user should be able to interrupt or skip it.
Do — respect prefers-reduced-motion. Disable bob, pulse, and spring; keep fades.
Don't — use bouncy / spring easing on functional UI (modals, sheets, menus). Reserve it for character & celebration.

Reduced motion

Honor the OS-level prefers-reduced-motion media query. When set:
  • Disable mascot bob, idle pulse, recording ring.
  • Replace slide transitions with crossfades (160ms).
  • Keep functional state animations (thinking dots, checkmark) but reduce duration by 30%.
